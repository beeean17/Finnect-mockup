# Finnect Guard 기능 구조 및 앱 워크플로우 정의서

> **목적**  
> 본 문서는 Finnect Guard의 최종 기능 구조를 정리하고, 스마트폰 뱅킹 앱 형태의 HTML 목업에서 구현 가능한 모든 주요 워크플로우를 정의하기 위한 기획 명세서이다.  
> 목업 제작 시 각 화면, 버튼, 위험도 분기, 사용자 선택 결과를 그대로 시뮬레이션할 수 있도록 작성하였다.

---

## 1. 서비스 정의

### 1.1 한 줄 정의

**Finnect Guard는 사용자가 대출, 카드 발급, 송금, 자동이체, 소비, 의심입금 대응 같은 금융 행동을 실행하기 전, 그 행동이 만들 비용·위험·책임을 AI가 쉬운 말로 보여주고, 위험하면 잠시 멈추게 하며, 문제 발생 시 신고기록까지 자동 정리하는 AI 금융 안전벨트 서비스이다.**

### 1.2 짧은 설명

> **누르기 전에 보여주고, 위험하면 멈추고, 문제 생기면 기록까지 남기는 AI 금융 안전벨트.**

### 1.3 핵심 문제의식

기존 금융 앱은 사용자가 **빠르게 가입하고, 빠르게 송금하고, 빠르게 결제하게 만드는 UX**에 집중한다.  
하지만 실제 금융 취약 사용자는 버튼을 누르기 전 다음을 충분히 이해하지 못한다.

- 이 대출을 받으면 매달 얼마를 갚아야 하는지
- 리볼빙을 신청하면 다음 달 이후 이자가 어떻게 붙는지
- 자동이체를 추가하면 월말 생활비가 부족해지는지
- 송금 상대와 계좌 명의가 다른 경우 어떤 위험이 있는지
- 모르는 사람이 보낸 입금을 바로 사용해도 되는지
- 배달·쇼핑·구독 결제가 이번 달 생활비를 침범하는지
- 사고가 났을 때 어떤 기록을 남겨야 하는지

Finnect Guard는 이러한 문제를 **금융 행동 직전 개입** 방식으로 해결한다.

## 2. 최종 확정 용어

| 구분 | 확정 용어 | 설명 |
|---|---|---|
| 서비스명 | **Finnect Guard** | 금융 행동을 보호하는 AI 안전벨트 |
| 서비스 설명 | **AI 금융 안전벨트** | 금융 버튼을 누르기 전 위험을 확인하고 멈추게 하는 보호 장치 |
| 핵심 시장 | **금융 행동 보호 시장** | 약관 요약, 금융교육, 사기 신고가 아니라 행동 직전 보호에 초점 |
| 기능 1 | **금융행동 위험진단** | 사용자가 하려는 금융 행동의 위험도를 AI가 분석 |
| 기능 2 | **쉬운말 핵심설명** | 어려운 약관과 금융 용어를 쉬운 말로 변환 |
| 기능 3 | **돈흐름 미리보기** | 버튼을 누른 뒤 1개월·3개월·최악 상황의 돈 흐름 예측 |
| 기능 4 | **생활비 보호모드** | 필수 생활비를 먼저 지키고 충동소비를 잠시 멈추게 함 |
| 기능 5 | **위험거래 잠시멈춤** | 고위험 송금, 의심입금, 리볼빙, 고금리 대출을 즉시 실행하지 않게 함 |
| 기능 5 하위 | **의심입금 안전확인** | 모르는 입금을 수락·보류·반환·신고 중 선택 |
| 기능 6 | **신고기록 자동정리** | 위험 경고, 사용자 선택, 대화·입출금 내역을 신고용 기록으로 정리 |
| MVP 이름 | **누르기 전 금융 안전확인** | 공모전용 MVP 명칭 |
| 대표 문장 | **누르기 전에 보여주고, 위험하면 멈추고, 문제 생기면 기록까지 남긴다.** | 발표용 핵심 문장 |

## 3. 최종 6대 기능 구조

| 번호 | 기능명 | 역할 | 사용자가 보는 문구 예시 |
|---:|---|---|---|

| 2 | **쉬운말 핵심설명** | 약관, 금융 용어, 수수료, 연체, 리볼빙 등을 쉬운 말로 설명 | “꼭 알아야 할 위험만 쉽게 설명할게요.” |

| 6 | **신고기록 자동정리** | 사고 발생 가능성이 있는 거래의 경고·선택·증거 기록을 자동 정리 | “문제가 생길 경우를 대비해 신고용 기록을 정리했어요.” |

## WF-00. 온보딩 및 보호 설정 워크플로우

### 목적
사용자의 금융 취약 지점과 보호 수준을 설정한다.

### 흐름

```mermaid
graph TD
    A["앱 첫 실행"] --> B["사용자 유형 선택"]
    B --> C["보호목표 선택"]
    C --> D["월 필수 생활비 입력"]
    D --> E["보호모드 설정"]
    E --> F["홈 화면 진입"]
```

### 화면 단계

| 단계 | 화면 | 사용자 입력 |
|---:|---|---|
| 1 | 시작 화면 | 서비스 설명 확인 |
| 2 | 사용자 유형 선택 | 사회초년생, 중고거래 사용자, 프리랜서, 고령층, 외국인 등 |
| 3 | 보호목표 선택 | 과소비 방지, 대출 위험 확인, 송금사기 예방, 의심입금 대응 |
| 4 | 생활비 입력 | 월세, 통신비, 교통비, 식비, 카드값, 대출상환액 |
| 5 | 보호모드 설정 | 쿨링오프 시간, 쉬운말 단계, 알림 강도 |
| 6 | 홈 진입 | 오늘 사용 가능 금액 확인 |

### 목업 버튼
- 시작하기
- 내 보호목표 고르기
- 생활비 입력하기
- 나중에 설정하기
- 보호모드 켜기

## WF-01. 금융행동 위험진단 기본 워크플로우

### 목적
사용자가 하려는 금융 행동을 AI가 인식하고 위험도를 카드로 보여준다.

### 흐름

```mermaid
graph TD
    A["위험 확인하기"] --> B["입력 방식 선택"]
    B --> C["화면 업로드"]
    B --> D["직접 입력"]
    B --> E["샘플 시나리오 선택"]
    C --> F["AI 분석"]
    D --> F
    E --> F
    F --> G["금융 행동 유형 분류"]
    G --> H["위험도 카드 생성"]
```

### 입력 유형

| 입력 방식 | 예시 |
|---|---|
| 화면 업로드 | 카드 발급 화면, 대출 신청 화면, 송금 화면, 입금 내역 캡처 |
| 텍스트 입력 | 약관 문장, 입금자명, 거래 메모, 금액 |
| 직접 선택 | 대출, 카드, 자동이체, 송금, 입금, 소비, 구독 |
| 샘플 실행 | 목업용 테스트 시나리오 선택 |

### 출력 카드

| 출력 항목 | 예시 |
|---|---|
| 금융 행동 | 소액대출 신청 |
| 위험도 | 위험 신호 있음 |
| 핵심 위험 | 월 상환 부담, 연체이자, 신용점수 하락 |
| 추천 행동 | 돈흐름 미리보기, 잠시멈춤, 금액 낮추기 |

## WF-02. 약관 위험진단 워크플로우

### 목적
긴 약관에서 사용자가 반드시 봐야 할 위험 조항만 추출한다.

### 흐름

```mermaid
graph TD
    A["약관 업로드"] --> B["위험 조항 추출"]
    B --> C["쉬운말 핵심설명"]
    C --> D["핵심 위험 카드"]
    D --> E["이해 완료"]
    D --> F["더 쉽게 설명"]
    D --> G["가입 보류"]
```

### 탐지 조항

| 조항 유형 | 위험 설명 |
|---|---|
| 연체이자 | 정해진 날에 못 갚으면 추가 비용 발생 |
| 리볼빙 | 카드값을 미루는 대신 이자 발생 |
| 중도상환수수료 | 대출을 빨리 갚아도 비용이 생길 수 있음 |
| 자동갱신 | 해지하지 않으면 계약이 계속 이어짐 |
| 개인정보 제3자 제공 | 내 정보가 다른 회사에 전달될 수 있음 |
| 실적 조건 | 혜택을 받으려면 매달 일정 금액 이상 써야 함 |

### 사용자 선택

# 15. 발표용 요약

## 15.1 기능 요약

> Finnect Guard는 **금융행동 위험진단 → 쉬운말 핵심설명 → 돈흐름 미리보기 → 생활비 보호모드 → 위험거래 잠시멈춤 → 신고기록 자동정리**의 6단계로 작동한다.
