# Finnect Guard 모바일 HTML 목업 시뮬레이션 명세서

> 목적: 이 문서는 **Finnect Guard** 기능을 스마트폰 뱅킹 앱 스타일의 HTML 목업으로 시각화하기 위한 상세 명세서이다.  
> 산출물 목표: 단일 `index.html` 파일만으로, 실제 은행 API 없이도 **약관 분석, 쉬운 설명, 트랜잭션 드라이런, Guard Hold, Safe Receive, 사고 대응 기록**의 모든 주요 경우를 클릭으로 시뮬레이션할 수 있게 한다.

---

## 0. HTML 생성용 전체 지시문

아래 내용을 기준으로 `index.html` 하나짜리 정적 프로토타입을 만든다.

- 스타일은 토스 같은 모바일 금융 앱 느낌으로 한다.
  - 흰색 또는 아주 밝은 회색 배경
  - 넓은 여백
  - 큰 숫자와 짧은 문장
  - 둥근 카드형 UI
  - 파란색 계열 포인트 컬러
  - 위험은 빨간색, 주의는 주황색, 안전은 초록색 계열로 표시
- 실제 토스 로고나 상표는 사용하지 않는다.
- 화면 크기는 스마트폰 기준으로 한다.
  - 권장 프레임: `390px x 844px`
  - 가운데 스마트폰 프레임을 두고, 내부에 앱 화면을 렌더링한다.
- 외부 API 없이 동작해야 한다.
- 외부 라이브러리 없이 순수 HTML, CSS, JavaScript로 구현한다.
- 모든 데이터는 JavaScript 내부의 샘플 데이터 객체로 둔다.
- 클릭 가능한 버튼, 탭, 칩, 모달, 바텀시트, 토스트 메시지를 구현한다.
- 사용자가 시나리오를 바꾸면 화면의 위험도, 설명, 드라이런 결과, 추천 행동, 증거 패키지가 바뀌어야 한다.
- 실제 송금, 실제 대출, 실제 신고는 수행하지 않는 **시뮬레이션 목업**임을 화면 하단에 작은 안내문으로 표시한다.

## 1. 서비스 한 줄 컨셉

**Finnect Guard는 사용자가 금융 버튼을 누르기 전, 그 버튼이 만들 미래의 비용·위험·책임을 미리 보여주고, 위험하면 멈추게 하며, 문제가 생기면 증거까지 남기는 AI 금융 행동 안전벨트다.**

## 2. 목업에서 반드시 보여줘야 하는 5대 기능

| 번호 | 기능명 | 핵심 역할 | 목업에서 보여줄 화면 |
|---:|---|---|---|
| 1 | AI 금융 행동 분석 카드 | 사용자가 하려는 금융 행동과 위험도를 카드로 요약 | 분석 결과 카드 화면 |
| 2 | 쉬운 금융 설명 오버레이 | 어려운 금융 용어와 약관을 쉬운 말로 설명 | 쉬운 설명 바텀시트 / 용어 카드 |
| 3 | 금융 트랜잭션 드라이런 | 1개월 뒤, 3개월 뒤, 최악의 상황을 예측 | 타임라인 시뮬레이션 화면 |
| 4 | Guard Hold / Safe Receive | 위험 행동을 보류·쿨링오프·수락·반환·신고로 분기 | 보류 화면, 의심입금 대응 화면 |
| 5 | 금융 사고 대응 기록 | 경고 이력과 사용자 선택을 증거 패키지로 저장 | 기록 타임라인, 증거 패키지 화면 |

## 3. 전체 화면 구조

HTML 목업은 다음 화면을 모두 포함한다.

| 화면 ID | 화면명 | 설명 |
|---|---|---|
| `screen-splash` | 스플래시 | Finnect Guard 로고와 한 줄 설명 |
| `screen-onboarding` | 사용자 유형 선택 | 사회초년생, 중고거래 사용자, 프리랜서, 고령층, 외국인 모드 선택 |
| `screen-home` | 홈 대시보드 | 오늘의 금융 위험, 빠른 분석, 최근 기록, 보호모드 상태 |
| `screen-input` | 금융 행동 입력 | 약관/대출/카드/송금/입금/자동이체/투자 시나리오 선택 |
| `screen-analyzing` | AI 분석 중 | 스캔 애니메이션과 분석 단계 표시 |
| `screen-risk-card` | AI 금융 행동 분석 카드 | 행동 유형, 위험도, 핵심 위험, 확인할 항목 표시 |
| `screen-easy-explain` | 쉬운 설명 오버레이 | 어려운 용어를 쉬운 말로 설명하고 이해 체크 |
| `screen-dryrun` | 트랜잭션 드라이런 | 1개월 뒤, 3개월 뒤, 최악의 상황 시뮬레이션 |
| `screen-guard-hold` | Guard Hold | 위험 거래 보류, 쿨링오프, 사유 입력 |
| `screen-safe-receive` | Safe Receive | 의심입금 수락/보류/반환/신고 선택 |
| `screen-evidence` | 증거 패키지 | 신고·상담용 증거 묶음 생성 |
| `screen-history` | 기록함 | 과거 분석, 보류, 신고자료 기록 리스트 |
| `screen-settings` | 보호 설정 | 보호모드, 고령층 모드, 외국어 모드, 금액 기준 설정 |

## 4. 전역 내비게이션

하단 탭 바는 다음 4개로 구성한다.

| 탭 | 라벨 | 이동 화면 |
|---|---|---|
| 1 | 홈 | `screen-home` |
| 2 | 분석 | `screen-input` |
| 3 | 기록 | `screen-history` |
| 4 | 설정 | `screen-settings` |

홈 화면 상단에는 다음 요소를 둔다.

- `Finnect Guard` 텍스트 로고
- 작은 상태 배지: `보호모드 ON`
- 알림 아이콘
- 현재 사용자 유형 배지: `사회초년생`, `중고거래`, `프리랜서`, `고령층`, `외국인`

| 사용자 유형 | 화면 표시명 | 추가 보호 포인트 | UI 변화 |

| 사회초년생 | 첫 금융 보호모드 | 카드, 대출, 리볼빙, 연체 설명 강화 | 기본 모드 |
| 중고거래 사용자 | 거래 안전모드 | 송금, 입금, 명의 불일치, 삼자사기 경고 강화 | 송금/입금 카드 우선 노출 |

## 7. 위험도 체계

목업은 모든 시나리오에 다음 위험도 체계를 적용한다.

| 위험도 | 점수 | 색상 | 화면 문구 | 기본 액션 |
|---|---:|---|---|---|
| 낮음 | 0~39 | 초록 | 지금 진행해도 큰 위험은 낮아 보여요 | 진행 가능 |
| 확인 필요 | 40~64 | 주황 | 몇 가지 확인 후 진행하는 게 좋아요 | 확인 후 진행 / 보류 |
| 높음 | 65~84 | 빨강 | 지금 바로 진행하면 손해나 분쟁이 생길 수 있어요 | Guard Hold |
| 긴급 | 85~100 | 진한 빨강 | 사기·연체·분쟁 위험이 큽니다. 지금 멈추는 걸 권장해요 | 보류 / 신고자료 정리 |

## 8. 기능 1: AI 금융 행동 분석 카드

### 8.1 카드 공통 구성

분석 결과 카드에는 다음 필드를 모두 표시한다.

| 필드 | 표시 예시 |
|---|---|
| 금융 행동 | 소액대출 신청 |
| 위험도 | 높음, 78점 |
| 한 줄 요약 | 매달 상환액이 월소득의 30%에 가까워요 |
| 핵심 위험 3개 | 월 상환 부담, 연체이자, 신용점수 영향 |
| 확인해야 할 것 | 실제 적용 금리, 중도상환수수료, 연체이자율 |
| 추천 행동 | 10분 보류 후 다시 확인하기 |
| 신뢰도 | AI 분석 신뢰도 86% |

### 8.2 시나리오별 분석 카드 문구

#### A. 대출 신청

| 상태 | 카드 문구 | 핵심 위험 | 추천 행동 |
|---|---|---|---|
| 안전 | 월소득 대비 상환 부담이 낮아요 | 중도상환수수료 확인 | 진행 가능 |
| 주의 | 매달 고정 상환액이 생활비를 압박할 수 있어요 | 월 상환액, 총 이자 | 금액 축소 검토 |
| 고위험 | 현재 소득 기준으로 연체 가능성이 높아 보여요 | 연체, 신용점수 하락, 추가 이자 | Guard Hold |

#### B. 카드 발급

| 상태 | 카드 문구 | 핵심 위험 | 추천 행동 |
|---|---|---|---|
| 안전 | 연회비와 실적 조건이 비교적 단순해요 | 연회비 | 진행 가능 |
| 주의 | 혜택을 받으려면 매월 일정 금액을 써야 해요 | 실적 조건, 과소비 유도 | 3개월 지출 확인 |
| 고위험 | 리볼빙 또는 결제 이월 위험이 있어요 | 이자, 연체, 신용 영향 | 리볼빙 해제 확인 |

#### C. 송금

| 상태 | 카드 문구 | 핵심 위험 | 추천 행동 |
|---|---|---|---|
| 안전 | 기존 거래 상대이고 계좌명도 일치해요 | 없음 | 진행 가능 |
| 확인 필요 | 처음 보내는 계좌예요 | 착오송금, 거래 이력 없음 | 상대 확인 |
| 고위험 | 거래 상대와 계좌 명의가 달라요 | 사기, 분쟁, 회수 어려움 | 보류 권장 |
| 긴급 | 급하게 돈을 보내라고 요구하고 있어요 | 피싱, 사기 | 신고자료 정리 |

#### D. 의심입금

| 상태 | 카드 문구 | 핵심 위험 | 추천 행동 |
|---|---|---|---|
| 안전 | 등록된 거래와 일치하는 입금이에요 | 없음 | 수락 |
| 확인 필요 | 모르는 사람의 입금이에요 | 착오입금, 분쟁 | 보류 |
| 고위험 | 거래 상대와 입금자명이 달라요 | 삼자사기, 통장협박 | 사용하지 말고 보류 |
| 긴급 | 입금 직후 다른 계좌로 반환을 요구했어요 | 사기 연루 가능성 | 원입금 기준 반환 문의 / 신고자료 정리 |

#### E. 자동이체

| 상태 | 카드 문구 | 핵심 위험 | 추천 행동 |
|---|---|---|---|
| 안전 | 월 고정 지출 증가가 크지 않아요 | 고정비 증가 | 진행 가능 |
| 주의 | 매월 빠져나가는 돈이 늘어나요 | 잔액 부족, 연체 | 다음 월 예산 확인 |
| 고위험 | 결제일에 잔액 부족 가능성이 높아요 | 연체, 수수료, 신용 영향 | 보류 |

#### F. 투자상품

| 상태 | 카드 문구 | 핵심 위험 | 추천 행동 |
|---|---|---|---|
| 안전 | 원금 보장형 상품으로 보여요 | 중도해지 이율 | 진행 가능 |
| 주의 | 해지 시 손해가 있을 수 있어요 | 중도해지, 수수료 | 기간 확인 |
| 고위험 | 원금손실 가능성이 있어요 | 손실, 환매 제한 | Guard Hold |

## 9. 기능 2: 쉬운 금융 설명 오버레이

### 9.1 오버레이 작동 방식

- 사용자가 `쉽게 설명해줘` 버튼을 누르면 바텀시트가 올라온다.
- 바텀시트는 다음 탭을 가진다.
  - `한 줄 설명`
  - `왜 위험한지`
  - `내가 확인할 것`
  - `예시로 보기`
- 사용자가 내용을 읽고 `이해했어요` 버튼을 누르면 이해 체크가 완료된다.

### 9.2 용어 사전

| 어려운 용어 | 쉬운 설명 | 위험 포인트 |
|---|---|---|
| 리볼빙 | 이번 달 카드값 일부를 다음 달로 미루는 기능 | 미룬 금액에 이자가 붙을 수 있음 |
| 연체이자 | 돈을 늦게 냈을 때 추가로 내는 이자 | 신용점수에 불이익 가능 |
| 중도상환수수료 | 대출을 일찍 갚을 때 내는 수수료 | 빨리 갚아도 비용이 생길 수 있음 |
| 자동갱신 | 기간이 끝나도 자동으로 다시 계약되는 것 | 원하지 않는 결제가 이어질 수 있음 |
| 제3자 제공 | 내 정보가 다른 회사에도 전달되는 것 | 마케팅, 심사, 제휴에 활용될 수 있음 |
| 원금손실 | 내가 넣은 돈보다 적게 돌려받을 수 있음 | 투자상품에서 중요 |
| 환매 제한 | 투자한 돈을 바로 빼기 어려운 조건 | 급히 돈이 필요할 때 불리함 |
| 실적 조건 | 혜택을 받기 위해 매월 써야 하는 금액 | 과소비를 유도할 수 있음 |
| 결제일 | 카드값이나 자동이체가 빠져나가는 날 | 잔액 부족 시 연체 위험 |
| 계좌 명의 불일치 | 거래 상대 이름과 계좌 주인 이름이 다른 상황 | 사기 가능성 증가 |
| 원입금 반환 | 돈이 들어온 경로로 되돌려 보내는 방식 | 다른 계좌로 보내면 분쟁 위험 증가 |
| 보류 | 지금 바로 처리하지 않고 멈춰두는 선택 | 실수와 충동을 줄임 |

### 9.3 보호모드별 설명 차이

| 모드 | 설명 방식 |
|---|---|
| 기본 | 핵심 위험 중심으로 간결하게 설명 |
| 고령층 | 한 화면에 한 문장, 큰 글씨, 반복 확인 |
| 외국인 | 쉬운 한국어 + 영어 보조 문구 |
| 사회초년생 | 신용점수, 연체, 리볼빙, 카드 실적 예시 강화 |
| 중고거래 | 명의 불일치, 삼자사기, 원입금 반환 설명 강화 |
| 프리랜서 | 세금계산서, 입금자명, 거래처, 정산 위험 설명 강화 |

## 11. 기능 4-A: Guard Hold

### 11.1 Guard Hold 발생 조건

다음 조건 중 하나 이상이면 Guard Hold를 띄운다.

| 조건 | 예시 |
|---|---|
| 위험도 65점 이상 | 대출 상환 부담 높음 |
| 고액 송금 | 사용자 설정 기준 초과 |
| 처음 보는 계좌 | 이전 거래 이력 없음 |
| 계좌 명의 불일치 | 거래 상대와 계좌명이 다름 |
| 리볼빙 또는 대출 | 이자와 연체 위험 존재 |
| 자동이체 후 잔액 부족 가능 | 결제일 잔액 부족 예상 |
| 고령층 보호모드 ON | 고액·고위험 거래는 항상 재확인 |
| 사용자가 급하게 진행 | 짧은 시간 안에 여러 번 확인 버튼 클릭 |

### 11.2 Guard Hold 화면 구성

- 상단 문구: `잠깐, 이 금융 행동은 한 번 더 확인이 필요해요`
- 위험 요약 3개
- 사유 입력란
  - placeholder: `지금 꼭 진행해야 하는 이유를 적어주세요`
- 쿨링오프 선택
  - `10분 뒤 다시 보기`
  - `내일 다시 보기`
  - `이번 달 예산 확인 후 진행`
- 버튼
  - `그래도 진행하기`
  - `보류하기`
  - `취소하기`
  - `상담/신고자료 만들기`

### 11.3 Guard Hold 상태별 문구

| 상황 | 문구 |
|---|---|
| 대출 고위험 | 이 대출은 매달 빠져나가는 돈이 커서 생활비를 압박할 수 있어요 |
| 카드 리볼빙 | 리볼빙은 결제를 미루는 대신 이자가 붙을 수 있어요 |
| 송금 고위험 | 거래 상대와 계좌 명의가 다릅니다. 송금 후 회수하기 어려울 수 있어요 |
| 자동이체 고위험 | 결제일에 잔액이 부족할 가능성이 있어요 |
| 투자 고위험 | 원금 손실 가능성이 있습니다. 여유자금인지 확인해 주세요 |

### 11.4 목업용 쿨링오프 동작

실제 10분을 기다리지 않고, 데모용으로 다음 버튼을 둔다.

- `타이머 시작`
- `데모용 10분 경과 처리`
- `보류 기록 보기`

타이머가 끝나면 다음 문구를 표시한다.

> 10분이 지났어요. 아직도 진행이 필요하다고 생각하면 다시 확인해 주세요.

## 12. 기능 4-B: Safe Receive

### 12.1 Safe Receive 발생 조건

| 조건 | 위험도 |
|---|---|
| 등록된 거래와 금액·입금자 일치 | 낮음 |
| 모르는 입금자 | 확인 필요 |
| 입금자명과 거래 상대명 불일치 | 높음 |
| 입금 후 다른 계좌로 반환 요구 | 긴급 |
| 소액 입금 후 신고 또는 계좌정지 협박 | 긴급 |
| 같은 금액이 반복 입금 | 확인 필요 |
| 거래 메모가 환불, 대여, 보증금 등 분쟁성 단어 포함 | 확인 필요 |

### 12.2 Safe Receive 화면 구성

- 상단 문구: `이 입금, 바로 써도 될까요?`
- 입금 정보 카드
  - 입금자명
  - 금액
  - 입금 시간
  - 메모
  - 연결된 거래 여부
- 위험 신호 리스트
- 선택 버튼 4개
  - `수락`
  - `보류`
  - `원입금 반환 안내`
  - `신고자료 정리`

### 12.3 선택지별 결과

| 선택 | 결과 화면 |
|---|---|
| 수락 | 정상 입금으로 기록, 홈으로 이동 |
| 보류 | 사용하지 말라는 안내, 보류 기록 생성 |
| 원입금 반환 안내 | 다른 계좌가 아닌 원래 입금 경로 기준으로 금융사 문의 안내 |
| 신고자료 정리 | 증거 패키지 생성 화면으로 이동 |

### 12.4 Safe Receive 시나리오별 문구

#### 안전 입금

> 등록된 거래와 입금자, 금액이 일치합니다. 정상 입금으로 보입니다.

#### 모르는 입금

> 처음 보는 입금자입니다. 실수 입금일 수 있으니 바로 사용하지 말고 확인해 주세요.

#### 삼자사기 의심

> 거래 상대와 입금자명이 다릅니다. 입금자가 아닌 다른 사람이 반환을 요구하면 위험할 수 있어요.

#### 통장협박 의심

> 소액 입금 후 신고나 계좌정지를 언급하는 경우, 즉시 사용하지 말고 기록을 남겨 상담을 준비하세요.

function handleReceiveAction(action) {

## 30. 최종 HTML 생성 프롬프트

아래 프롬프트를 그대로 사용하면 HTML 목업을 생성할 수 있다.

```text
Finnect Guard라는 모바일 금융 안전 앱의 정적 HTML 목업을 만들어줘.

디자인은 토스 같은 스마트폰 뱅킹 앱 느낌으로 하되 실제 토스 로고나 상표는 쓰지 말고, 흰 배경, 큰 카드, 둥근 모서리, 파란 포인트 컬러, 넓은 여백을 사용해줘.

단일 index.html 파일로 작성하고 외부 라이브러리 없이 HTML, CSS, JavaScript만 사용해줘. 화면은 390px x 844px 스마트폰 프레임 안에 들어가야 해.

앱은 다음 5대 기능을 모두 시뮬레이션해야 해.
1. AI 금융 행동 분석 카드
2. 쉬운 금융 설명 오버레이
3. 금융 트랜잭션 드라이런
4. Guard Hold / Safe Receive
5. 금융 사고 대응 기록

하단 탭은 홈, 분석, 기록, 설정으로 구성해줘.

분석 화면에서는 사용자가 다음 시나리오를 선택할 수 있게 해줘.
- loan_safe
- loan_caution
- loan_high
- card_safe
- card_caution
- card_high_revolving
- terms_privacy
- terms_auto_renewal
- transfer_safe
- transfer_caution
- transfer_high_mismatch
- transfer_high_urgent
- receive_safe
- receive_unknown
- receive_high_thirdparty
- receive_high_blackmail
- autopay_safe
- autopay_caution
- autopay_high
- saving_safe
- investment_high

각 시나리오마다 위험도, 위험 점수, 핵심 위험, 쉬운 설명, 1개월 뒤 결과, 3개월 뒤 결과, 최악의 상황, 추천 행동이 다르게 나오게 해줘.

위험도는 낮음, 확인 필요, 높음, 긴급 4단계로 보여줘.
낮음은 초록, 확인 필요는 주황, 높음은 빨강, 긴급은 진한 빨강으로 표시해줘.

위험도가 높은 대출, 카드, 송금, 자동이체, 투자상품은 Guard Hold 화면으로 이동해야 해.
Guard Hold 화면에는 사유 입력, 10분 보류, 취소, 그래도 진행하기 버튼을 넣어줘.
10분 보류는 실제 10분을 기다리지 않고 데모용으로 '10분 경과 처리' 버튼을 넣어줘.

의심입금 시나리오는 Safe Receive 화면으로 이동해야 해.
Safe Receive 화면에는 수락, 보류, 원입금 반환 안내, 신고자료 정리 버튼을 넣어줘.
신고자료 정리를 누르면 증거 패키지 화면으로 이동해줘.

증거 패키지 화면에는 사건 요약, 거래 정보, 위험 신호, 사용자 선택 기록, 추천 대응을 보여줘.
기록함에는 사용자가 선택한 진행, 보류, 취소, 신고자료 생성 기록이 리스트로 남아야 해.

설정 화면에는 보호모드 ON/OFF, 고령층 큰 글씨 모드, 쉬운 한국어/외국인 모드, 고액 송금 기준 설정을 넣어줘.
고령층 모드를 켜면 글자와 버튼 크기를 키워줘.
외국인 모드를 켜면 주요 문구에 영어 보조 라벨을 작게 표시해줘.

모든 화면 하단에는 '본 화면은 공모전 목업이며 실제 금융거래를 수행하지 않습니다.'라는 안내 문구를 작게 표시해줘.
```
